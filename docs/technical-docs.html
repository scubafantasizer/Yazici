1. Abstract

Yazıcı is an autonomous production engine designed to bridge the gap between abstract human intent and concrete executable code. This document details the architectural benchmarks and operational standards of the v3.0.0 release.

2. System Architecture

The system utilizes a Production Line model, where every stage of development is optimized for throughput and contextual accuracy.

Component System Logic Implementation
Duct Complexity Routing Multi-tier provider selection (Gemini/Claude/GPT)
Ink Context Serialization Selective "Hot Brick" contextualization
Nozzle Execution Layer Transactional filesystem operations
Bed Persistence C++20 Optimized SQLite Persistence

3. Persistence and Native Performance

Persistence is managed by better-sqlite3, compiled with the -std=c++20 flag. This enables high-velocity transaction logging and state recovery, essential for long-running high-workload sessions.

4. ESM Compliance

Yazıcı is built on a pure ESM foundation. It relies on import.meta.url and fileURLToPath for robust path resolution across diverse environments, ensuring that the "nozzle" always points exactly where the intent implies.

5. Security Protocol

Secrets and API keys are vaulted within an encrypted SQLite table. Access is restricted via a secure API layer, preventing accidental leakage during the printing process.
